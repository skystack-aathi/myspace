SkyStack:

A Modern Data Platform for Secure Advertising

Introduction

This project was designed for an advertising company handling highly sensitive customer and banking data, where security, governance, and reliability were critical. The goal was to build a scalable, automated, and compliant data platform that supports both operational and analytical workloads while ensuring strict access control.

Azure Data Ecosystem & Governance

The Azure ecosystem was chosen to leverage its robust cloud-native capabilities, scalability, and strong enterprise security features. To ensure data protection and governance, Unity Catalog and RBAC were implemented. This allowed centralized control over data access, lineage tracking, and secure data sharing across teams, which is essential when handling sensitive financial information.

  • Configured Unity Catalog for centralized governance, lineage, and metadata management
  • Implemented RBAC to enforce role-based access across users and teams
  • Ensured secure handling of sensitive banking and customer data
  • Enabled controlled and auditable data access across the platform

Data Ingestion & File Processing

To handle incoming data efficiently, Apache NiFi was used for real-time routing and processing of files received from SharePoint. NiFi was chosen for its ability to manage data flows visually and handle priority-based processing, which is critical in advertising workflows where certain campaigns require faster processing.

Additionally, Azure Data Factory (ADF) was used to orchestrate scheduled and periodic data ingestion, ensuring reliable and automated movement of data into Azure Data Lake Storage.

  • Processed incoming files from SharePoint into production paths
  • Used Apache NiFi for priority-based queue processing and routing
  • Configured alert mechanisms to notify teams on high-priority data processing
  • Implemented ADF pipelines for scheduled and automated ingestion
  • Ingested and stored data into Azure Data Lake Storage (ADLS)

Data Vault Modeling & Warehouse Design

The Data Vault model was chosen due to its scalability, flexibility, and ability to handle historical data efficiently. This approach allows the system to adapt to changing business requirements without impacting existing structures, making it ideal for complex and evolving datasets.

  • Designed and implemented Hubs, Links, and Satellites for structured modeling
  • Ingested transformed data into Azure SQL / SQL Server
  • Maintained historical tracking and auditability of all data changes
  • Enabled scalable and modular data architecture

Data Transformation & Analytics

For transformation, dbt (data build tool) was used to create modular, testable, and reusable transformation pipelines. This improved maintainability and ensured consistency across transformation logic. Azure Synapse Analytics was leveraged for high-performance querying and analytical processing, supporting business reporting such as client contribution analysis.

  • Implemented transformations using dbt
  • Processed and analyzed data using Azure Synapse Analytics
  • Created data models for reporting and business intelligence
  • Supported client contribution analytics and quarterly reporting

Reliability, Monitoring & Automation

To ensure reliability, data pipelines were designed with fault tolerance in mind. Raw and processed data are stored in Amazon S3, enabling reprocessing in case of failures. Automated monitoring and alerting systems were implemented to notify teams proactively, reducing downtime and ensuring faster issue resolution.

  • Stored raw and intermediate data in Amazon S3
  • Enabled reprocessing of failed pipelines from stored files
  • Implemented alerts and email notifications for failures
  • Reduced manual intervention and improved issue resolution time

Orchestration & DevOps

To manage complex workflows, Apache Airflow is being used for orchestration, ensuring proper dependency management across pipelines. Azure DevOps handles CI/CD and production deployments, enabling automated and reliable releases.

  • Orchestrating workflows using Apache Airflow
  • Managing CI/CD pipelines with Azure DevOps
  • Automating deployments and pipeline execution
  • Ensuring production-grade reliability and scalability

Impact & Results

  • Reduced manual effort by approximately 80% through automation
  • Improved data processing performance by 20–30%
  • Ensured secure and compliant handling of sensitive data
  • Delivered consistent and reliable data for business reporting

💡 Outcome: A secure, scalable, and automated data platform that ensures high data quality, strong governance, and reliable insights for business decision-making.
